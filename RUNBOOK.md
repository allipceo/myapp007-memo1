# MyApp007 - Runbook (장애 대응 매뉴얼)

본 앱은 정적 HTML 기반이며, 발생 가능한 주요 장애와 복구 절차는 다음과 같습니다.

## 1. localStorage 데이터 조작/손상 (JSON Parse Error)
- **증상**: 페이지 로드 시 "내용이 없습니다"라는 메시지가 반복되거나 콘솔에 `SyntaxError`가 발생함.
- **수단**: 앱 내에 자동 복구 로직이 포함되어 있음. JSON 파싱 실패 시 상단에 안내 문구가 뜨며 저장소가 빈 배열 `[]`로 자동 초기화됨.
- **수동 복구**: 브라우저 개발자 도구(F12) -> Application -> Local Storage -> `memos` 키를 수동으로 Delete.

## 2. 저장 용량 초과 (QuotaExceededError)
- **증상**: 메모 저장 시 "저장 공간이 부족하거나 저장에 실패했습니다"라는 빨간색 안내 문구가 표시됨.
- **원인**: 브라우저의 localStorage 용량 제한(보통 5MB)에 도달함.
- **대응**: 불필요한 기존 메모를 삭제하여 공간을 확보한 후 다시 저장을 시도함.

## 3. 특정 특수문자 표시 오류 (XSS 의심)
- **증상**: `<script>` 등의 문자열을 입력했을 때 코드가 실행되는 것처럼 보임.
- **확인**: 본 앱은 모든 출력을 `textContent`로 처리하므로 코드가 '실행'될 수 없음. 단순히 텍스트 그대로 보인다면 정상임.
- **대응**: 만약 팝업(alert)이 뜬다면 즉시 코드 내 `innerHTML` 사용 지점을 찾아 `textContent`로 교체해야 함.
