## 0) 사용법 (이 문서가 ‘행동 순서’ 그 자체)

### v3.1 운영체계 — 초기 LLM 기획 → 3자 구도 전환(잠금)

- **초기 기획(대단위 조사/브레인스토밍/종합):** LLM 활용
- **3자 구도 운영:** 조대표–선과장–개발팀장(안팀장 또는 고팀장)
- **전환(스위치) 기준:** Gate 1~7이 전부 승인 완료(잠금 완료)되면, 이후부터는 3자 구도로만 운영한다.
- **스위치 이후 원칙:** “아이디어 발산”이 아니라 **체크리스트 통과 게임(구현/검증/증거)** 으로만 진행한다.
- **스코프 팽창 차단:** 스위치 이후 신규 아이디어/기능추가는 즉시 반영 금지. 반드시 **차기 버전 백로그로 격리**한다.
- **역할(고정):**
    - 조대표: 결정/승인(OK 문구 + 스코프 변경 승인)
    - 선과장: 문서·DB·지시·검토·판정(게이트 운영/감사/증거요건 강제)
    - 개발팀장: 구현·실측·증거(커밋/스냅샷/재현/워크스루)
- 이 페이지는 ‘체크리스트 모음’이 아니라, **프로젝트를 실제로 진행하는 순서**대로 설계된 실행 템플릿이다.
- 규칙: 각 단계는 **작성(잠금값) → 승인(기록) → 실행(산출물/증거)** 순으로만 진행한다.
- v3.1 핵심: 초기 “대단위 조사/브레인스토밍/종합”은 LLM으로 수행하되, Gate 1~7이 승인 완료(잠금 완료)되면 **3자(조대표–선과장–개발팀장)** 구도로 전환하여 실행한다.
- v3.0 핵심: 기능/스키마보다 앞에 **Gate 0(실기기/부팅/환경 잠금)** 을 둔다.

### ⏱️ Gate 시간 기록(품질/속도 KPI) — 자동 준수 절차

- 원칙: 각 Gate는 “검토 개시 시간”과 “승인 시간”을 반드시 기록한다.
- 정의(잠금):
    - **검토 개시 시간** = *초안이 올라와서 리뷰 가능한 상태가 된 시점*
    - **승인 시간** = 1차 승인/최종 승인 “OK”가 기록된 시점
- 역할(잠금): **작성자 기록 → 1차 승인자 → 최종 승인자(오너)**
- 금지: 기록 없이 다음 Gate로 이동(= Gate 생략으로 간주)

아래 블록을 각 Gate 섹션 상단(또는 승인 콜아웃 바로 위)에 복붙하여 반드시 채운다:

- [ ]  검토 개시: YYYY-MM-DD HH:MM (TZ) / 작성: ___
- [ ]  1차 승인: OK/보류 (YYYY-MM-DD HH:MM)
- [ ]  최종 승인: OK/보류 (YYYY-MM-DD HH:MM)
- [ ]  리드타임: 개시→1차 ***분, 1차→최종*** 분, 총 ___분

### 🧩 Gate Q&A 운영 방식(잠금) — “각 Gate 제목 바로 아래 `Q&A (Gate n)` 토글 세트에서만 진행”

- 원칙: 각 Gate에서 발생하는 모든 Q&A는 **해당 Gate 섹션의 맨 상단(게이트 제목 바로 아래)** `Q&A (Gate n)` 토글 세트에서만 진행한다.
- 역할(고정): **개발팀장(질문/작업결과 보고) → 선과장(답변/1차 승인) → 조대표(최종 승인)**
- 구조(고정):
    - 질문/보고 (개발팀장/작성자)
    - 답변 (선과장/리뷰어)
    - 선과장 1차 승인 (이 Q&A)
    - 조대표 최종 승인 (이 Q&A)
- 기록 규칙:
    - 질문/답변은 한 줄 요약 + 필요 시 본문/하위 bullet로 증거를 남긴다.
    - 승인 문구/시간은 해당 Gate의 “승인 기록(콜아웃)”에도 최종 반영한다.
- 효과: 가독성↑ + 의사결정/승인 추적 가능

## 시나리오 — MyApp007 메모앱

- 플로우 1 — 메모 작성
    - 입력: 텍스트 입력 후 "저장" 버튼 클릭
    - 기대출력: 메모가 목록에 추가되고 localStorage에 저장됨
    - 실패 시 처리: 빈 입력으로 저장 시도 → 저장 거부 + "내용을 입력하세요" 안내 표시
- 플로우 2 — 메모 목록 조회
    - 입력: 페이지 로드 (또는 새로고침)
    - 기대출력: localStorage에 저장된 메모 전체가 목록에 표시됨
    - 실패 시 처리: 저장된 메모 없음 → "메모가 없습니다" 빈 상태 메시지 표시
- 플로우 3 — 메모 삭제
    - 입력: 메모 항목의 "삭제" 버튼 클릭
    - 기대출력: 해당 메모가 목록에서 지워지고 localStorage에서도 제거됨
    - 실패 시 처리: 없음 (삭제는 항상 성공)

## 예외/엣지케이스(최소 5개)

1. 빈 메모 저장 시도 → 저장 거부 + 안내 문구 표시

5. 브라우저 시크릿 모드 → localStorage 무효, 범위 외(일반 모드 사용 안내)
